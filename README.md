# Описание

Данный проект представляет собой Telegram-бота, отвечающий на вопросы студентов,
с помощью RAG-системы. Ответы основаны на внутренних документах НИУ ВШЭ.

# Архитектура

1. Подготовка данных (emb.py, SemanticChuncker.py)

Используем подготовленную заранее базу данных .docx файлов.
Класс SemanticChunker разбивает текст на абзацы, вычисляет эмбенддинги. 
Параметры чанкинга могут настраиваться.
К модели эмбенддингов Alibaba-NLP/gte-multilingual-base, добавляем префиксы "document: " и "query: " для улучшения качества поиска.
Векторная база данных сохраняется локально в файлы .faiss и в дальшейшем используется ботом.

2. Векторный поиск (bot.py)

Бот отправляет запрос в векторную базу данных и получает ответ, строит эмбенддинг с префиксом "query: " под запрос пользователя.
Параметры поиска: 
* k = 5 - число возвращаемых чанков
* fetch_k = 15 - количество предварительно отобранных кандидатов
* lambda_mult = 0.7 – баланс между релевантностью и разнообразием

3. Генерация ответа (bot.py)

Используется модель Qwen/Qwen2.5-7B-Instruct, которая загружается локально.
Функция build_rag_prompt собирает промпт для генерации ответа (контекст + вопрос).
Генерация выполняется в отдельном потоке и ограничена 70 токенами.

4. Telegram-бот (bot.py)

Используется фреймворк AsyncTeleBot. 
Существующие команды:
* /start - вывод сообщения-приветствия с описанием функционала бота.
* /help - вывод справки о том как пользоваться ботом с описанием команд.
* /mail - запуск настройки рассылок (только для администраторов). Предлагается выбрать категории студентов и написать сообщение рассылки.
* /prompt - команда-инструмент для ответов на вопросы. После этой команды пользователь может задать вопрос, который будет отправлен в пайплайн генерации ответов.

Выводятся сообщения об ошибках при векторном поиске и генерации ответа.

# Стек 

1. Язык: Python 3.10+
2. Telegram API: AsyncTeleBot
3. Обработка документов: docx2txt, langchain-community
4. Эмбеддинги: HuggingFaceEmbeddings (модель Alibaba-NLP/gte-multilingual-base)
5. Векторная база данных: FAISS
6. Генерация: Qwen2.5-7B-Instruct, transformers
7. Память: torch, gc, bitsandbytes

# User story

## При получении ответа на вопросы

1. Роль: Студент МИЭМ НИУ ВШЭ
2. Действие: Задать вопрос об организации учебного процесса
3. Предусловие: Пользователь зарегистрирован в мессенджере Telegram и нашел бота по юзернейму.
4. Запускающее событие: Команда /start
5. Сценарий: 
* Пользователь вводит свой вопрос
* Пользователь получает ответ через короткое время 
6. Альтернативный сценарий:
* Пользователь вводит свой вопрос
* Пользователь получает сообщение об ошибке
* Пользователь ждет некоторое время или связывается с тех. поддержкой
* Повторение основного сценария
7. Ценность: Быстро получить точный ответ, без необходимости искать информацию вручную или спрашивать у учебного офиса.

## При отправке рассылок

1. Роль: Сотрудник учебного офиса
2. Действие: Отправить рассылку студентам
3. Предусовие: Пользователь зарегистрирован в мессенджере Telegram и зарегистрирован в боте как администратор
4. Запускающее событие: Команда /mail
5. Сценарий: 
* Пользователь выбирает категории студентов и нажимает кнопку "Подтвердить категории"
* Пользователь вводит сообщение для рассылки и нажимает кнопку "Подтвердить"
7. Ценность: Удобный и быстрая настройка рассылок
