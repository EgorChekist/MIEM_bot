# Benchmark для оценки качества RAG-бота

Benchmark нужен для проверки качества ответов бота по документам НИУ ВШЭ.
Он позволяет сравнивать разные модели и настройки по одному и тому же набору вопросов.

## Основные файлы

- `benchmark.csv` — набор тестовых вопросов и критериев.
- `run_benchmark.py` — скрипт для запуска benchmark.
- `results/` — папка для результатов прогонов моделей.

## Структура benchmark.csv

Основные колонки:
- `id` — номер вопроса;
- `category` — тема вопроса;
- `subtopic` — подтема;
- `question` — вопрос для модели;
- `question_type` — тип вопроса;
- `expected_answer_summary` — ожидаемый смысл ответа;
- `source_doc` — документ-источник;
- `must_contain` — что хороший ответ должен содержать;
- `should_not_contain` — чего в ответе быть не должно;
- `notes` — комментарии.

## Как оценивать ответы

Оценка ставится вручную по шкале:
- 5 — хороший и правильный ответ, без выдумок;
- 4 — в целом правильно, но не хватает деталей;
- 3 — частично правильно;
- 2 — слабый ответ;
- 1 — очень плохой ответ;
- 0 — галлюцинация, ответ не по документам.

## Что проверяет benchmark

Benchmark проверяет весь RAG-пайплайн.
Если ответ плохой, причина может быть в:
- поиске не того документа;
- слабой модели;
- плохом prompt.
